# Capstone: Enterprise AIOps and Amazon Q Business

## Project Overview

Design and implement an enterprise AIOps platform that integrates Amazon Q Business for organizational knowledge access, CloudShell-based operational automation, and a governed MLOps pipeline with cost controls. The platform demonstrates production-grade AI operations including Bedrock API automation, cost monitoring with SageMaker resource management, and enterprise RAG workflows with console prototyping.

## Deliverables

### 1. Amazon Q Business Deployment with CloudShell Automation
Deploy an Amazon Q Business instance connected to enterprise data sources, and build a CloudShell-based automation toolkit for AI-assisted CLI operations. Create reusable shell scripts for common Bedrock operations including model invocation, resource listing, and cost monitoring. Demonstrate Amazon Q capabilities for document search, summarization, and question answering across organizational data.

### 2. Enterprise MLOps Framework with Cost Controls
Implement an MLOps framework that manages the lifecycle of AI models from experimentation through deployment, including automated cost controls for SageMaker resources and Bedrock provisioned throughput. Build CloudShell scripts for monitoring and cleaning up expensive AI resources, and demonstrate understanding of diminishing returns in LLM scaling through a cost-effectiveness analysis of different model sizes.

### 3. Enterprise AIOps Dashboard and RAG Workflow
Build an enterprise AIOps implementation combining Bedrock RAG workflows for knowledge retrieval with console-based prototyping for rapid model experimentation. Create a governance report documenting the complete AIOps architecture including Amazon Q integration, CloudShell automation, MLOps lifecycle management, and cost control policies. Include measurable metrics for operational efficiency improvements.

## Evaluation Criteria

### Distinction
- Amazon Q deployment demonstrates measurable improvements in knowledge access across at least two enterprise data sources
- MLOps framework includes automated cost alerting and resource cleanup with documented savings analysis
- AIOps dashboard provides actionable operational metrics with governance policies for AI resource management

### Proficient
- Amazon Q is operational with CloudShell automation scripts for common Bedrock operations
- MLOps framework manages model deployment with basic cost monitoring and SageMaker resource tracking
- RAG workflows produce grounded responses with console prototyping demonstrating model comparison

### Developing
- Amazon Q or CloudShell automation is functional but not both integrated into a unified workflow
- MLOps framework exists but lacks cost controls or automated resource management
- RAG workflow operates but governance documentation is incomplete
