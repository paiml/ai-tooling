# Capstone: CLI Automation with Amazon Q and CloudShell

## Project Overview

Build a complete CLI automation toolkit that leverages Amazon Q and CloudShell to streamline AWS development workflows, from Docker container management and CDK infrastructure deployment to Rust development and ECR container registry operations. The toolkit demonstrates AI-assisted infrastructure-as-code practices with end-to-end serverless deployment pipelines.

## Deliverables

### 1. Amazon Q CloudShell Automation Toolkit
Create a reusable toolkit of CloudShell scripts that automate common AWS operations using Amazon Q for AI-assisted CLI command generation. Build Docker container workflows that run entirely within CloudShell, including image building, testing, and ECR push operations. Demonstrate Amazon Q integration for generating and debugging Docker commands, AWS CLI operations, and troubleshooting deployment issues.

### 2. CDK Serverless Deployment Pipeline
Build an end-to-end serverless deployment pipeline using AWS CDK with Amazon Q assistance in CloudShell. Deploy Lambda functions using CDK stacks, configure API Gateway endpoints, and automate the complete lifecycle from CDK synthesis through deployment. Include a Rust development workflow in CloudShell that compiles and tests Rust Lambda functions with Amazon Q-generated code.

### 3. Container Registry Workflow with Architecture Documentation
Implement a complete Docker-to-ECR container workflow from CloudShell, including multi-stage Docker builds, image tagging strategies, ECR authentication, and automated push operations. Produce architecture diagrams documenting the complete container pipeline from source code through CloudShell build to ECR registry and production deployment. Include a comparison of manual versus AI-assisted workflow efficiency.

## Evaluation Criteria

### Distinction
- Automation toolkit includes at least ten reusable CloudShell scripts covering Docker, CDK, and ECR operations
- CDK pipeline deploys a multi-resource stack with Lambda, API Gateway, and monitoring from CloudShell
- Architecture documentation includes measured efficiency improvements from AI-assisted versus manual workflows

### Proficient
- CloudShell scripts automate Docker and ECR operations with Amazon Q integration
- CDK deploys at least one Lambda function with basic infrastructure-as-code practices
- Container workflow is end-to-end from build to ECR push with architecture diagrams

### Developing
- CloudShell automation covers basic operations but lacks Amazon Q integration or reusability
- CDK deployment is partial or uses console instead of infrastructure-as-code
- Container workflow is functional but lacks architecture documentation or efficiency analysis
