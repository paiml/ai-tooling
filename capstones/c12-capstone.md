# Capstone: Multi-Modal AI: Screenshots to Production Code

## Project Overview

Build a web application using multi-modal AI development techniques, progressing from screenshot-to-code conversion through iterative refinement with visual feedback to automated browser testing with MCP and Playwright. The project demonstrates the complete multi-modal development lifecycle including prompt engineering with visual context, GitHub Copilot integration, and documentation-driven iteration.

## Deliverables

### 1. Screenshot-to-Code Application with Visual Iteration
Convert a set of design screenshots into a working web application using multi-modal AI tools (GitHub Copilot, VS Code). Apply prompt engineering with visual context to guide code generation, using screenshots as primary input combined with textual instructions. Iterate on the generated code through at least three refinement cycles using image-based visual feedback, documenting how each iteration improves fidelity to the original design.

### 2. Documentation-Driven Development Workflow
Implement a documentation-driven development workflow where project requirements, component specifications, and design decisions are written as documentation before code generation. Use documentation as AI context to improve code generation accuracy. Set up live reload for rapid feedback between AI generation and visual verification. Document the challenges encountered during iteration including regression, context drift, and diminishing returns, with strategies for each.

### 3. MCP and Playwright Automation Pipeline
Build an automated validation pipeline using Model Context Protocol (MCP) to connect AI tools with Playwright browser automation. Implement automated checks that validate generated code against design requirements by capturing screenshots and comparing visual output. Create end-to-end tests that verify the application functions correctly across multiple browsers. Produce a retrospective comparing multi-modal AI development productivity against traditional development for the same application scope.

## Evaluation Criteria

### Distinction
- Application achieves high visual fidelity to design screenshots with documented per-iteration improvement metrics
- Documentation-driven workflow demonstrates measurable improvement in AI code generation accuracy
- MCP-Playwright pipeline automates visual regression testing with quantified development productivity gains

### Proficient
- Screenshot-to-code produces a functional application with at least three visual iteration cycles
- Documentation guides AI code generation with live reload feedback loop operational
- Playwright tests validate basic application functionality with MCP integration

### Developing
- Initial screenshot-to-code conversion produces partial results without systematic visual iteration
- Documentation exists but is not demonstrated to improve AI generation accuracy
- Browser automation is basic without MCP integration or visual regression testing
