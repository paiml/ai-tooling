# Capstone: AI-Powered Analytics and Performance Engineering

## Project Overview

Build an AI-powered analytics platform that combines Rust-Bedrock integration for intelligent data analysis, Lambda performance benchmarking comparing Rust and Python runtimes, and AI-assisted code transformation pipelines. The platform demonstrates measurable performance and cost improvements through systematic benchmarking, energy efficiency analysis, and Amazon Q-driven analytics workflows.

## Deliverables

### 1. Rust-Bedrock Analytics Pipeline with Code Transformation
Build a Rust application that connects to Amazon Bedrock for AI-powered analytics, including a code instrumentation pipeline that adds performance monitoring to existing code using generative AI. Implement an AI-assisted Python-to-Rust code transformation pipeline that converts analytics workloads and measures the performance improvement. Document the architecture with integration diagrams.

### 2. Lambda Benchmarking and Cost Analytics Suite
Create a systematic Lambda benchmarking suite that compares Rust and Python implementations across cold start time, memory usage, execution duration, and per-invocation cost. Use Claude through Bedrock to analyze benchmark results and generate insights. Integrate SageMaker DataWrangler for data preparation and produce a quantitative cost comparison report that demonstrates the business case for language migration.

### 3. Amazon Q Analytics and Energy Efficiency Report
Deploy Amazon Q for AI-powered business intelligence with living insights from enterprise data. Configure CodeCatalyst development environments for analytics projects and demonstrate Amazon Q's ability to translate analytics workflows between languages. Produce an energy efficiency analysis of AI analytics workloads that compares resource consumption across different implementation strategies and documents sustainable computing practices.

## Evaluation Criteria

### Distinction
- Code transformation pipeline produces working Rust code from Python with measured performance gains exceeding 5x
- Benchmarking suite includes at least five Lambda configurations with statistical analysis of results
- Energy efficiency report includes quantitative resource consumption data with actionable optimization recommendations

### Proficient
- Rust-Bedrock analytics pipeline is functional with basic code transformation capability
- Lambda benchmarks compare Rust and Python with documented cost and performance differences
- Amazon Q analytics and energy efficiency analysis are completed with supporting data

### Developing
- Analytics pipeline connects to Bedrock but lacks code transformation or instrumentation
- Benchmarks are limited to a single configuration without systematic comparison methodology
- Energy efficiency analysis is qualitative without quantitative resource measurement
