# Capstone: Build a Production SaaS Application with AI

## Project Overview

Build and launch a complete SaaS application from architecture through deployment and marketing, using AI-assisted development techniques at every stage. The application includes a FastAPI backend with API key authentication, Docker containerization with CI/CD automation, a marketing site deployed on GitHub Pages, and developer documentation for API consumers.

## Deliverables

### 1. API-Driven SaaS Application with Testing
Plan and build a production API using FastAPI and Python, starting from MVP requirements through data exploration, endpoint definition, and API documentation. Implement a comprehensive test harness using pytest that validates all API endpoints. Leverage LLMs for code generation and architecture planning while applying engineering judgment to verify AI-generated output. Deliver a working API with auto-generated interactive documentation.

### 2. Containerized Deployment Pipeline
Containerize the application using Docker with a production-ready Dockerfile. Build a CI/CD pipeline that automatically builds, tests, and publishes container images to a container registry. Implement API key authentication to control and meter access to the service. Demonstrate the complete container lifecycle from local development through automated build to registry publication, with verification at each stage.

### 3. Marketing Site and Developer Documentation
Create a converting landing page, pricing page with tiered plans, and about page for the SaaS product. Deploy the marketing site to GitHub Pages for zero-cost hosting. Write developer documentation that covers authentication, endpoint reference, code examples, and onboarding instructions. Produce a launch readiness assessment evaluating the product across technical completeness, marketing presence, and monetization readiness.

## Evaluation Criteria

### Distinction
- API includes at least five endpoints with comprehensive pytest coverage and interactive documentation
- CI/CD pipeline automates build, test, and publish with API key authentication and usage metering
- Marketing site achieves conversion-focused design with complete developer documentation and pricing strategy

### Proficient
- API is functional with test harness and basic documentation
- Docker container builds and publishes to a registry with basic CI/CD automation
- Landing page and developer documentation are deployed with at least two pricing tiers

### Developing
- API runs locally but lacks automated testing or comprehensive documentation
- Containerization is manual without CI/CD pipeline or registry publication
- Marketing presence or developer documentation is incomplete
