# Capstone: AI-Assisted Debugging and Test-Driven Fixes

## Project Overview

Apply AI-assisted debugging techniques to a real codebase with known defects, using test-driven development to systematically isolate, reproduce, and fix bugs. The project combines structured context gathering, logging-assisted AI diagnosis, and proactive bug discovery to demonstrate the complete AI debugging lifecycle from bug identification through verified fix.

## Deliverables

### 1. Test-Driven Bug Resolution Portfolio
Select a codebase with at least five known bugs of varying severity. For each bug, follow the test-driven debugging workflow: write a failing test that reproduces the bug, use AI tools to analyze the failure and suggest fixes, implement the fix, and verify the test passes. Document the before-and-after for each bug including the test case, AI-generated analysis, and the applied fix. Demonstrate that AI suggestions require verification by identifying at least one case where the AI-generated fix was incorrect or incomplete.

### 2. Context-Driven Debugging Toolkit
Build a debugging toolkit that systematically gathers context before querying AI tools. Implement structured logging that produces output consumable by both humans and AI debugging assistants. Demonstrate the context-logging-AI debugging workflow on at least two complex bugs, showing how gathered context improves AI diagnosis accuracy compared to undirected queries. Include architecture documentation that maps code organization to debugging entry points.

### 3. Proactive Bug Discovery Report
Use AI tools to proactively scan a codebase for potential unknown bugs, including hidden defects that have not yet manifested as failures. Document the AI-driven bug discovery process, categorize findings by severity and type, and validate at least three discoveries through test cases. Produce a report comparing AI-assisted proactive bug hunting against traditional code review for defect detection rate and false positive frequency.

## Evaluation Criteria

### Distinction
- Bug resolution portfolio includes at least five bugs with complete test-driven workflows and documented AI hallucination cases
- Context toolkit demonstrates measurable improvement in AI diagnosis accuracy with structured logging versus unstructured queries
- Proactive discovery validates at least three genuine bugs with false positive analysis and detection rate comparison

### Proficient
- At least three bugs are resolved using the test-driven debugging workflow with AI assistance
- Context gathering and logging are implemented with at least one demonstrated improvement in AI diagnosis
- Proactive scanning identifies potential bugs with at least one validated through testing

### Developing
- Bugs are fixed but without systematic test-driven workflow or AI verification analysis
- Context gathering exists but impact on AI diagnosis quality is not measured
- Proactive scanning is attempted but findings lack validation through test cases
